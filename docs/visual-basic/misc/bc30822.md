# <a name="nullconstant-is-not-declared"></a>' \<nullconstant> ' não está declarado
' \<nullconstant> ' não está declarado. Não há mais suporte para constantes nulas; em vez disso, use System. DBNull.  
  
 Uma instrução usa a `Null` palavra-chave, que não tem mais suporte no Visual Basic.  
  
 **ID do erro:** BC30822  
  
## <a name="to-correct-this-error"></a>Para corrigir este erro  
  
1. Use <xref:System.DBNull> em vez de `Null`. O exemplo a seguir demonstra isso.  
  
    ```vb  
    Sub TestDBNull()  
        Dim t As DataTable  
        ' Assume the DataGrid is bound to a DataTable.  
        t = CType(DataGrid1.DataSource, DataTable)  
        Dim r As DataRow  
        r = t.Rows(datagrid1.CurrentCell.RowNumber)  
        r.BeginEdit  
        r(1) = System.DBNull.Value ' Assign DBNull to the record.  
        r.EndEdit  
        r.AcceptChanges  
        If r.IsNull(1) Then  
            MsgBox("")  
        End If  
    End Sub  
    ```  
  
2. Use a palavra-chave [Nothing](../language-reference/nothing.md) para atribuições e comparações quando você usar variáveis de objeto. O exemplo a seguir demonstra isso.  
  
    ```vb  
    Sub TestNothing()  
        Dim cls As Object  
        ' cls is Nothing if it has not been assigned using the New keyword.  
        If (cls Is Nothing) Then  
            MsgBox("cls is Nothing")  
        End If  
        cls = Nothing ' Assign Nothing to the class variable cls.  
    End Sub  
    ```
